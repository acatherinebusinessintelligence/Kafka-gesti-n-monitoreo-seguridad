PostgreSQL con Podman Compose

Integración con Kafka Connect (JDBC Sink) — guía paso a paso.

Podman PostgreSQL Kafka Connect WSL2

¿Cómo encaja con Kafka?

Flujo: Topic Kafka → Kafka Connect (JDBC Sink) → PostgreSQL. El conector lee del tópico y escribe filas en la BD. Como es relacional, necesita tipos ⇒ usamos mensajes con esquema (JSON con schema/payload) o Avro/Protobuf.

[ productores ]  --->  (topic: ordersschema)  --->  [ Kafka Connect - JDBC Sink ]  --->  [ PostgreSQL: tabla orders ]

Prerrequisitos: clúster Kafka activo (zookeeper, kafka1, kafka2, kafka3) y todos en la misma red (normalmente kafka-lab_default).

podman ps --format "{{.Names}}  {{.Networks}}  {{.Status}}"
NET=$(podman inspect zookeeper -f '{{range $k,$v := .NetworkSettings.Networks}}{{$k}}{{end}}')
echo "$NET"   # debería ser kafka-lab_default

Nombre del tópico: usamos ordersschema (sin guion bajo) para evitar el warning de colisión ._ en métricas.

1) Archivo podman-compose-postgres.yml

Guárdalo en tu carpeta del lab. Usa la misma red externa del clúster (kafka-lab_default).

version: "3.8"

services:
  postgres:
    image: docker.io/library/postgres:15
    container_name: postgres
    hostname: postgres
    environment:
      POSTGRES_DB: demo
      POSTGRES_USER: demo
      POSTGRES_PASSWORD: demo
    ports:
      - "5432:5432"
    volumes:
      - ./pgdata:/var/lib/postgresql/data
      # Descomenta para crear objetos la primera vez:
      # - ./initdb:/docker-entrypoint-initdb.d:ro
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $$POSTGRES_USER -d $$POSTGRES_DB"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - kafka-lab_default

  # (Opcional) UI web para administrar Postgres
  pgadmin:
    image: docker.io/dpage/pgadmin4:8.10
    container_name: pgadmin
    hostname: pgadmin
    depends_on:
      - postgres
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@example.com
      PGADMIN_DEFAULT_PASSWORD: admin123
    ports:
      - "8081:80"
    networks:
      - kafka-lab_default

networks:
  kafka-lab_default:
    external: true

Nota: Si la red no existe: podman network create kafka-lab_default.

2) Levantar Postgres (+pgAdmin opcional)

# Sólo la base
podman-compose -f podman-compose-postgres.yml -p kafka-lab up -d postgres

# Con pgAdmin también
podman-compose -f podman-compose-postgres.yml -p kafka-lab up -d pgadmin

# Verifica
podman ps --format "{{.Names}}  {{.Networks}}  {{.Status}}  {{.Ports}}"

pgAdmin: http://localhost:8081 (usuario admin@example.com / clave admin123).

3) Probar conectividad desde Kafka

podman exec -it kafka1 bash -lc 'getent hosts postgres || echo NO-RESUELVE'

4) Preparar Kafka Connect

4.1 Instalar plugin JDBC

podman exec -it connect bash -lc 'confluent-hub install --no-prompt confluentinc/kafka-connect-jdbc:latest && \
  (grep -q "^plugin.path=" /etc/kafka/connect-distributed.properties || \
   echo "plugin.path=/usr/share/java,/usr/share/confluent-hub-components" >> /etc/kafka/connect-distributed.properties)'
podman restart connect
curl -s http://localhost:8083/connector-plugins | grep -i jdbc

4.2 Asegurar driver JDBC de Postgres

podman exec -it connect bash -lc 'cd /usr/share/confluent-hub-components/confluentinc-kafka-connect-jdbc/lib && \
  (ls -1 | grep -i postgresql- || curl -LO https://jdbc.postgresql.org/download/postgresql-42.7.4.jar)'
podman restart connect

5) Crear tópico y enviar mensajes con esquema

# Crear topic sin underscore
podman exec -it kafka1 bash -lc \
  "kafka-topics --create --if-not-exists --topic ordersschema \
   --partitions 3 --replication-factor 3 --bootstrap-server kafka1:29092"

# Archivo con 3 eventos (JSON con schema/payload)
cat > ordersschema.jsonl <<'JSONL'
{"schema":{"type":"struct","name":"orders_value","optional":false,"fields":[
  {"field":"order_id","type":"int32","optional":false},
  {"field":"customer","type":"string","optional":false},
  {"field":"amount","type":"double","optional":false}
]},"payload":{"order_id":1001,"customer":"ana","amount":59.9}}
{"schema":{"type":"struct","name":"orders_value","optional":false,"fields":[
  {"field":"order_id","type":"int32","optional":false},
  {"field":"customer","type":"string","optional":false},
  {"field":"amount","type":"double","optional":false}
]},"payload":{"order_id":1002,"customer":"luis","amount":120.0}}
{"schema":{"type":"struct","name":"orders_value","optional":false,"fields":[
  {"field":"order_id","type":"int32","optional":false},
  {"field":"customer","type":"string","optional":false},
  {"field":"amount","type":"double","optional":false}
]},"payload":{"order_id":1003,"customer":"eva","amount":18.5}}
JSONL

# Enviar sin modo interactivo
cat ordersschema.jsonl | podman exec -i kafka1 bash -lc \
  "kafka-console-producer --bootstrap-server kafka1:29092 --topic ordersschema"

8) Script útil: create_topic.sh

Para crear tópicos sin escribir podman exec cada vez:

#!/bin/bash
# Uso: ./create_topic.sh <topic_name> [partitions] [replication]
PARTITIONS=${2:-3}
REPLICATION=${3:-3}
BROKER="kafka1:29092"

if [ -z "$1" ]; then
  echo "Uso: $0 <topic_name> [partitions] [replication]"
  exit 1
fi

TOPIC=$1
echo "Creando tópico '$TOPIC' con $PARTITIONS particiones y RF=$REPLICATION..."

podman exec -it kafka1 bash -lc \
  "kafka-topics --create --if-not-exists \
   --topic $TOPIC --partitions $PARTITIONS \
   --replication-factor $REPLICATION --bootstrap-server $BROKER"

podman exec -it kafka1 bash -lc \
  "kafka-topics --describe --topic $TOPIC --bootstrap-server $BROKER"

9) Problemas comunes

  • Driver faltante: “No suitable driver for jdbc:postgresql” → instala el JAR y reinicia Connect.
  • Sin esquema: “non-null Struct schema” → usa JSON con schema/payload.
  • Mensaje truncado: JsonEOFException → recrea el tópico y produce desde archivo.
  • Red: NO-RESUELVE → misma red externa del lab.
  • Credenciales: “password authentication failed” → valida user/clave y DB demo.
